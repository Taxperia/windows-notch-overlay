# Code of Conduct

## Our Standards

This project should stay practical, respectful, and useful for people building or testing Windows desktop tooling.

Examples of expected behavior:

- Be direct and constructive.
- Assume good intent when reviewing issues and pull requests.
- Keep criticism focused on code, behavior, documentation, or design tradeoffs.
- Respect different skill levels and operating environments.
- Avoid harassment, insults, threats, sexualized language, and personal attacks.

## Scope

This code of conduct applies in project spaces, including issues, pull requests, discussions, release comments, and any other community channel connected to this repository.

## Reporting

If you see behavior that violates this code of conduct, report it through the repository owner or GitHub's abuse reporting tools. Include links, screenshots, and context when possible.

Maintainers may remove comments, close discussions, block users, or take other moderation actions when behavior harms the project.

## Enforcement

Enforcement should be proportional and focused on keeping the project safe and productive. Repeated or severe violations may lead to a permanent ban from project spaces.
